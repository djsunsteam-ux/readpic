# Readpic for Mac — UI 风格规范

> **版本:** v0.1  
> **日期:** 2026-05-14  
> **适用范围:** Readpic v1 UI / UX 设计与实现  
> **关联文档:** `PRD.md`, `TECH_SPEC.md`, `ROADMAP.md`

---

## 1. 设计方向

Readpic 的 UI 风格采用：

> **原生极简 Pro + 少量现代轻量元素**

核心目标是让 Readpic 看起来像一款认真打磨过的 macOS 原生工具：轻、快、克制、稳定，图片永远是视觉中心。

### 1.1 关键词

- 原生
- 极简
- 专注
- 轻量
- 高级灰
- 图片优先
- 低干扰
- 工具感
- 长期耐看

### 1.2 设计原则

1. **图片优先**  
   UI 不应抢图片的注意力。图片查看区域永远是主角。

2. **默认克制**  
   默认只显示必要操作。更多信息通过状态栏、信息面板、右键菜单或快捷键进入。

3. **符合 macOS 习惯**  
   优先使用系统控件、SF Symbols、系统颜色、系统动效和系统全屏行为。

4. **轻量但不简陋**  
   界面要干净，但空状态、信息面板、设置页需要有细节和完成度。

5. **动效服务于响应感**  
   动效要快、短、自然，不做炫技转场。

6. **深色优先，但浅色完整可用**  
   图片查看器的深色体验更重要，但浅色模式不能只是附带实现。

## 3. 色彩规范

### 3.1 色彩策略

Readpic 不建立复杂品牌色系统，优先使用 macOS 系统语义色，并辅以少量固定背景色。

强调色默认使用系统蓝：

- macOS system accent color 优先
- 默认蓝色参考：`#0A84FF`

### 3.2 深色模式

| 用途 | 色值 | 说明 |
|---|---|---|
| 图片画布背景 | `#0B0B0D` | 默认深色背景，接近黑但不死黑 |
| 全屏背景 | `#000000` | 全屏沉浸浏览时使用纯黑 |
| 面板背景 | `#161618` | 信息面板、侧栏 |
| 工具栏背景 | `rgba(28, 28, 30, 0.72)` | 可配合 vibrancy |
| 状态栏背景 | `#111113` | 低干扰底栏 |
| 主文字 | `#F2F2F7` | 高对比文本 |
| 次级文字 | `#A1A1AA` | 文件信息、辅助文本 |
| 弱文字 | `#6E6E73` | 占位说明、禁用状态 |
| 分割线 | `rgba(255,255,255,0.08)` | 尽量弱化 |
| 危险色 | `#FF453A` | 删除、错误 |
| 收藏色 | `#FFD60A` | 收藏、星标 |

### 3.3 浅色模式

| 用途 | 色值 | 说明 |
|---|---|---|
| 图片画布背景 | `#F4F4F5` | 默认浅色背景 |
| 面板背景 | `#FFFFFF` | 信息面板、设置页 |
| 工具栏背景 | `rgba(255,255,255,0.72)` | 可配合 vibrancy |

| 次级文字 | `#636366` | 文件信息、辅助文本 |

| 白色 | `#FFFFFF` | `#FFFFFF` | 检查透明或白底图片 |

### 4.1 字体

- UI 字体：San Francisco / `.system`
- 等宽信息：SF Mono / `.monospacedDigit`

### 4.2 字号

| 场景 | 字号 | 字重 |
|---|---:|---|
| 工具栏按钮文字 | 13 | Regular / Medium |
| 状态栏文字 | 12 | Regular |
| 信息面板标题 | 13 | Semibold |
| 信息面板内容 | 12 | Regular |
| 空状态标题 | 20 | Semibold |
| 空状态说明 | 13 | Regular |
| 设置页分组标题 | 13 | Semibold |
| Toast | 13 | Regular |

### 4.3 数字显示

以下内容建议使用等宽数字，避免跳动：

- EXIF 参数

```text

```

## 7. 主界面规范

### 7.1 图片画布 Image Canvas

图片画布是主视觉区域。

要求：

- 背景纯净，无纹理。
- 图片居中显示。
- 默认适应窗口。
- 透明图片可显示棋盘格背景，但默认不启用；通过工具栏或设置切换。
- 图片阴影只在浅色背景下轻微使用。

浅色模式图片阴影建议：

```text
shadow color: rgba(0, 0, 0, 0.18)
shadow radius: 16
shadow y: 4
```

深色模式默认不加明显阴影。

### 7.2 顶部工具栏

工具栏风格：

- macOS 原生 toolbar 优先。
- 查看模式下保持轻量。
- 全屏或沉浸模式下可自动隐藏。
- 图标优先，必要时显示文字。

推荐工具栏内容：

```text
Grid | Fit | Zoom Out | Zoom In | Rotate Left | Rotate Right | Mirror | Thumbnails | Info
```

> **注意:** Open、100%、Finder 按钮已在 v1 工具栏中移除，移至菜单栏实现。详见下节 §7.5。

视觉要求：

- 不使用高饱和背景。
- 当前激活状态使用系统强调色或轻背景高亮。
- 禁用状态透明度降低。

### 7.3 底部状态栏

状态栏用于显示当前图片的核心信息。

默认内容：

```text
12 / 340    IMG_2048.HEIC    4032 × 3024    42%    3.8 MB
```

要求：

- 高度：24–28pt。
- 字号：12pt。
- 信息过长时中间截断文件名。
- 不使用强边框。
- 可以在设置中隐藏。

### 7.4 浮动操作反馈

缩放、加载、删除等临时反馈使用轻量浮层。

示例：

```text
100%
```

```text
已移到废纸篓    撤销
```

要求：

- 居中或底部居中。
- 背景使用半透明深色或系统 material。
- 显示时间 1.5–3 秒。
- 不阻塞用户操作。

---

### 7.5 菜单栏规范

Readpic 菜单栏遵循 macOS 标准布局，所有功能模块在菜单中均有对应入口。

#### 7.5.1 菜单结构

```text
File                          Edit                    View
├── Open Image… (⌘O)          ├── Copy Image (⌘C)     ├── Grid View (G)
├── Open Folder… (⌘⇧O)        ├── Copy File (⌘⇧C)     ├── ───────
├── ───────                   └── Copy File Path       ├── Fit Window
├── Close Window (⌘W)             (⌘⌥C)               ├── Fit Width
├── ───────                                          ├── Actual Size
├── Move to Trash (⌘⌫)                               ├── ───────
├── ───────                                           ├── Zoom In (⌘=)
├── Reveal in Finder (⌘⌥E)                           ├── Zoom Out (⌘-)
└── Open Externally (⌘E)                             ├── Reset Zoom (⌘0)
                                                     ├── ───────
Image                           Help                  ├── Thumbnail Strip
├── Rotate Left (⌘[)           └── Keyboard           ├── Info Panel (I)
├── Rotate Right (⌘])               Shortcuts (?)     ├── ───────
└── Flip Horizontal (⌘⇧H)                            ├── Sort By ▸
                                                     ├── Show Status Bar
                                                     ├── ───────
                                                     └── Fullscreen (F)
```

#### 7.5.2 设计原则

- **完整覆盖**：所有功能必须同时出现在菜单栏中，不允许仅有工具栏或快捷键而无菜单项。
- **macOS 惯例**：File 放文件操作、Edit 放剪贴板、View 放显示相关、Image 放图片变换。
- **快捷键一致**：菜单栏快捷键与 toolbar tooltip 及快捷键帮助浮层保持同步。
- **禁用状态**：无图片加载时，File / View / Image 中依赖当前文件的操作项自动 `disabled`。
- **Phase 2 扩展**：Crop、Resize、Rate、Filter By、Search 等 Phase 2 功能在各自所属菜单组中添加入口。

## 8. 空状态设计

### 8.1 默认空状态

首次启动或未打开图片时显示：

```text
          Readpic

   Drop images or folders here

      [ Open Image ]
      [ Open Folder ]

   Supports JPEG, PNG, HEIC, WebP, GIF, TIFF, BMP
```

中文：

```text
          Readpic

       拖入图片或文件夹

        [ 打开图片 ]
        [ 打开文件夹 ]

   支持 JPEG、PNG、HEIC、WebP、GIF、TIFF、BMP
```

### 8.2 视觉要求

- 内容居中。
- 图标可以使用轻量图片图标或应用图标。
- 标题不要过大。
- 按钮使用系统按钮样式。
- 支持拖拽时显示高亮边框或背景。

### 8.3 拖拽悬停状态

当用户拖入图片或文件夹时：

- 画布背景轻微变亮。
- 中央显示 “释放以打开”。
- 使用系统强调色描边。

## 9. 信息面板规范

### 9.1 结构

信息面板位于右侧，宽度建议 280–340pt。

分组：

```text
File
  Name
  Path
  Size
  Created
  Modified

Image
  Dimensions
  Format
  Color Space
  Bit Depth (Phase 1)

Camera
  Date Taken
  Camera
  Lens
  ISO
  Aperture
  Shutter
```

### 9.2 视觉风格

采用轻卡片分组，但保持 macOS 原生克制感。

卡片要求：

- 圆角 10pt。
- 背景比面板背景略高一层。
- 不使用强阴影。
- 分组标题 13pt Semibold。
- 内容 12pt Regular。

### 9.3 行布局

推荐 key-value 布局：

```text
Dimensions       4032 × 3024
Format           HEIC
Color Space      Display P3
Size             3.8 MB
```

要求：

- key 使用次级文字。
- value 使用主文字。
- 长路径支持复制，显示时中间截断。

## 10. 缩略图网格规范

### 10.1 网格视图

缩略图网格用于快速浏览当前文件夹。

要求：

- 默认缩略图尺寸：150–200pt（`GridItem(.adaptive(minimum: 150, maximum: 200))`）。
- 可调范围（Phase 2+）：96–320pt。
- 缩略图圆角：6pt。
- 文件名显示在缩略图下方。
- 当前选中项使用系统强调色描边。

### 10.2 缩略图状态

| 状态 | 表现 |
|---|---|
| 加载中 | 灰色占位块 + 文件名 |
| 加载失败 | 文件图标 + 错误提示 |
| 当前图片 | 强调色描边 |
| 多选 | 勾选标记 + 强调色背景 |
| 悬停 | 轻微背景高亮 |

### 10.3 占位图

占位图要低干扰：

- 深色：`#1C1C1E`
- 浅色：`#E5E5EA`
- 可显示简单图片图标。

## 11. 全屏与沉浸模式

### 11.1 全屏背景

全屏默认使用纯黑：

```text
#000000
```

### 11.2 UI 显示规则

- 鼠标移到顶部区域（距顶部 40px 内）时显示工具栏，移出后立即淡出。
- 鼠标移到底部区域（距底部 26px 内）时显示状态栏。
- 工具栏淡入淡出使用 150ms easeInOut。
- 底部缩略图条与工具栏同步显示/隐藏。

### 11.3 全屏动效

- 遵循系统全屏动画。
- 工具栏显示/隐藏：120–180ms。
- 图片切换默认无动画或极轻微 crossfade。

## 13. Toast 与提示规范

### 13.1 Toast 类型

| 类型 | 示例 |
|---|---|
| 成功 | 已复制图片 |
| 可撤销操作 | 已移到废纸篓 · 撤销 |
| 错误 | 文件损坏，无法显示 |
| 警告 | 此格式暂不支持 |
| 加载 | 正在加载 iCloud 文件 |

### 13.2 视觉

- 背景使用 system material 或半透明深色。
- 圆角 10pt。
- 文字 13pt。
- 最长显示 3 秒。
- 有操作按钮时可显示 5 秒。

### 13.3 文案原则

- 短句。
- 先说结果，再给操作。
- 不使用技术错误码作为主提示。

示例：

```text
已移到废纸篓    撤销
```

```text
不支持 CR3 格式    用外部应用打开
```

## 14. 动效规范

### 14.1 时长

| 场景 | 时长 |
|---|---:|
| 工具栏淡入淡出 | 120–180ms |
| Toast 出现/消失 | 160–220ms |
| 信息面板展开 | 180–240ms |
| 缩略图加载出现 | 120–180ms |
| 图片切换 | 0–120ms |
| 缩放 | 跟手，无额外延迟 |

### 14.2 动效原则

- 缩放和平移必须跟手。
- 图片切换不做明显滑动动画，避免影响快速翻页。
- 大文件加载不使用夸张 loading，只显示轻量进度或占位。
- 动效不能阻塞操作。

---

## 15. 可访问性

### 15.1 对比度

- 主文字必须满足常规可读性。
- 弱文字不能用于关键操作。
- 危险操作必须有颜色以外的文字说明。

### 15.2 键盘操作

- 所有核心功能必须可通过键盘完成。
- 焦点状态要清晰。
- `Esc` 始终可关闭浮层或退出当前模式。

### 15.3 VoiceOver

v1 至少为关键按钮提供 accessibility label：

- 打开图片
- 网格视图
- 适应窗口
- 旋转
- 信息
- 删除
- Finder 中显示

---

## 16. 文案风格

### 16.1 语气

- 简洁
- 直接
- 不打扰
- 不使用过度营销语

### 16.2 中文文案

推荐：

```text
打开图片
打开文件夹
拖入图片或文件夹
正在加载…
已复制图片
已移到废纸篓
不支持此格式
文件损坏，无法显示
```

避免：

```text
哎呀，出错了！
超棒！图片复制成功！
该文件由于未知原因无法被本程序解析
```

### 16.3 英文文案

推荐：

```text
Open Image
Open Folder
Drop images or folders here
Loading…
Image copied
Moved to Trash
Unsupported format
The file is damaged and can’t be displayed
```

## 18. 实现注意事项

### 18.1 优先使用系统能力

- SF Symbols
- system colors
- system material
- native toolbar where appropriate
- system full screen
- native context menu

### 18.2 避免过度自定义

v1 不做：

- 自绘复杂窗口 chrome
- 大量非系统图标
- 高饱和品牌渐变
- 花哨切图动画
- 类移动端的大圆角和重阴影

### 18.3 保持性能优先

任何视觉效果如果影响：

- 打开速度
- 翻页流畅度
- 缩放跟手程度
- 大文件夹滚动

则应降级或移除。

## 19. 后续可扩展方向

当 v1 稳定后，可以逐步增强品牌感：

- 设计专属 App Icon。
- 空状态加入更精致的插画或图标。
- README 使用统一截图风格。
- 增加轻量启动欢迎页。
- 为收藏、评分、对比模式建立更完整的视觉系统。

但主界面仍应坚持：**图片优先，UI 克制。**
