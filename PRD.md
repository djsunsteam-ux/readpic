## 1. 产品概述

### 1.1 产品名称

**Readpic** — 一款极速、轻量、原生体验优先的 macOS 图片查看器。

### 1.2 产品定位

Readpic 面向 macOS 用户，目标是替代系统「预览」在图片浏览场景下的体验短板，提供更快的文件夹浏览、更顺滑的缩放/翻页、更好的缩略图体验和更直接的文件操作。

v1 的核心目标不是一次性复刻 PicView 的所有功能，而是先做出一个稳定、快速、可日常使用的原生图片查看器。

### 1.3 核心卖点

- **极速打开:** 常见图片格式秒开，不阻塞 UI。
- **流畅浏览:** 键盘、鼠标、触控板操作自然顺滑。
- **大文件夹友好:** 支持大量图片文件夹的渐进式缩略图加载。
- **原生体验:** 遵循 macOS 交互习惯，支持全屏、拖拽、Finder 操作、深浅色模式。
- **开源分发:** 通过 GitHub Release 分发，面向愿意接受开源软件安装方式的用户。

## 2. 目标用户

### 2.1 普通用户

- 想要比「预览」更快地浏览同一文件夹内的大量图片。
- 经常查看截图、聊天图片、网页长图、下载图片。

### 2.2 设计师 / 开发者

- 快速查看设计稿、界面截图、素材图。
- 需要复制图片、复制文件路径、查看尺寸和基础元数据。

### 2.3 摄影爱好者

- 快速浏览照片文件夹。
- 初步筛选、删除、收藏候选照片。

---

## 3. 用户旅程

### 3.1 快速查看截图

用户双击一张图片，Readpic 打开图片并自动加载同目录图片列表。用户用方向键切换图片，用滚轮或触控板查看细节，按 `Esc` 关闭窗口。

### 3.2 浏览大量照片

用户拖入一个照片文件夹，Readpic 立即显示文件列表和占位缩略图，后台逐步生成缩略图。用户可以立即翻页、进入全屏、删除废片或在 Finder 中显示文件。

### 3.3 查看设计稿

用户拖入 PNG/WebP/HEIC 设计稿，Readpic 以适应窗口方式打开。用户可以放大到 100%、查看图片尺寸、复制图片或复制文件路径。

### 3.4 查看长截图

用户打开长图后，Readpic 默认优先适应宽度，滚轮默认缩放图片；到达顶部或底部后，可按方向键切换上一张/下一张。

## 4. v1 范围

### 4.1 v1 必须支持的格式

| 格式 | 优先级 | 说明 |
|---|---:|---|
| JPEG / JPG | P0 | 最核心格式 |
| PNG | P0 | 截图、设计稿常见格式 |
| HEIC / HEIF | P0 | macOS / iPhone 常见格式 |
| WebP | P0 | 网络图片常见格式 |
| GIF | P0 | v1 支持基础动画播放 |
| TIFF | P0 | macOS 常见专业/扫描格式 |
| BMP | P0 | 基础兼容 |
| ICO | P1 | 可延后，但实现成本较低 |

### 4.2 v1 核心功能

#### F-001 图片打开与显示

- 支持通过以下方式打开图片：
  - 双击文件
  - `⌘+O` 文件选择器
  - 拖拽图片到窗口
  - 拖拽文件夹到窗口
  - 从 Finder「打开方式」进入
- 打开单张图片时，自动扫描同目录支持格式图片。
- 图片默认使用「适应窗口」显示。
- 支持 100% 原始尺寸查看。
- 支持长图适应宽度。

#### F-002 文件夹浏览

- 支持当前文件夹内上一张/下一张浏览。
- 默认不递归扫描子文件夹。
- 支持按文件名自然排序（默认）。
- 支持按修改时间排序（可通过设置或菜单切换）。
- 大文件夹扫描不能阻塞 UI。

#### F-003 导航与缩放

- 键盘：
  - `←` 上一张
  - `→` 下一张
  - `Space` 动画暂停/继续（仅 GIF 动画时有效）
  - `Esc` 关闭浮层、退出当前模式或关闭窗口
- 鼠标：
  - 滚轮默认缩放；可在设置中切换为滚动平移或翻页
  - 双击切换适应窗口 / 100%
- 触控板：
  - 双指捏合缩放
  - 双指滚动平移
  - 双指左右滑动可切换图片（仅在滚轮行为为「翻页」时生效）
  - `?` 快捷键帮助浮层

#### F-004 缩略图网格

- 支持当前文件夹缩略图网格。
- 缩略图渐进加载，未完成时显示占位图和文件名。
- 支持虚拟滚动。
- 支持单选。
- 多选（`⌘+点击`、`Shift+点击`）延后到 Phase 2。
- 拖拽框选延后到 v2。

#### F-005 全屏与沉浸浏览

- 支持系统全屏。
- 全屏下支持翻页、缩放、退出、显示/隐藏工具栏。
- 鼠标移到窗口顶部显示工具栏。
- 底部可显示轻量缩略图条。

#### F-006 基础信息面板

v1 信息面板显示：

- 文件名
- 文件路径
- 文件大小
- 创建/修改时间
- 图片尺寸
- 格式
- 色彩空间
- 基础 EXIF：拍摄时间、相机型号、镜头、ISO、光圈、快门

完整 IPTC、XMP、GPS 地图、直方图延后。

#### F-007 基础文件操作

- `⌘+C` 复制图片到剪贴板。
- `⌘+Shift+C` 复制文件。
- `⌘+⌥+C` 复制文件路径。
- `⌘+⌥+E` 在 Finder 中显示。
- `⌘+E` 用外部应用打开。
- `⌘+Delete` 移到废纸篓。
- 移到废纸篓后显示 Toast，并提供「撤销」。
- 永久删除不作为 v1 默认快捷操作。

#### F-008 旋转与翻转

- 支持顺时针 / 逆时针旋转显示。
- 支持水平翻转显示。
- v1 默认仅改变当前查看状态；保存修改需要用户显式确认。

#### F-009 界面与状态栏

- 顶部工具栏：打开、网格、适应窗口、100%、旋转、信息、Finder（按 UI_STYLE_GUIDE §7.2）。
- 底部状态栏：
  - 当前序号 / 总数
  - 文件名
  - 图片尺寸
  - 缩放比例
  - 文件大小
  - 加载状态
- 空状态页面：
  - 拖入图片或文件夹
  - 打开文件
  - 打开文件夹
  - 打开最近文件夹（设置中启用「记住上次打开文件夹」后显示）

#### F-010 设置

v1 设置项：

- 主题：跟随系统 / 浅色 / 深色
- 背景色：跟随主题 / 纯黑 / 深灰 / 浅灰 / 白色 / 自定义
- 滚轮行为：滚动 / 翻页 / 缩放
- 默认缩放模式：适应窗口 / 适应宽度 / 100%
- 是否记住上次打开文件夹
- 是否显示状态栏

- Live Photos 视频部分播放
- 专业图像编辑

- 批量重命名
- 批量格式转换
- 批量 resize
- 直方图
- 颜色拾取器

- Mac App Store 版本
- App Sandbox 版本

这些功能可进入后续路线图，详见 `ROADMAP.md`。

## 6. 用户体验原则

### 6.1 快于功能多

如果某个功能会明显影响打开速度、翻页流畅度或内存占用，v1 优先不做。

### 6.2 不阻塞 UI

文件扫描、缩略图生成、图片解码、外部文件变化检测都必须后台执行。主界面应始终可响应。

### 6.3 不做意外破坏性操作

- 修改扩展名不自动转换图片格式。
- 删除默认进入废纸篓，不默认永久删除。
- 覆盖原图前必须明确确认。
- 旋转、翻转、裁剪等修改必须区分「仅显示」与「保存到文件」。

### 6.4 新用户无需读文档

打开、翻页、缩放、全屏、删除、查看信息等核心操作应能通过工具栏或浮层找到。

├── ───────                     ├── Rotate Right (⌘])

### Phase 2 菜单扩展

| 菜单 | 新增项 |
|---|---|
| File | Open Recent ▸、Export / Save As…、Batch Convert…、Batch Rename… |
| Edit | Select All (⌘A)、Copy Metadata |

## 8. 非功能需求

### 8.1 性能目标

| 指标 | v1 目标 |
|---|---:|
| 应用冷启动 | < 1s |
| 常见 1080p JPEG/PNG 热缓存打开 | < 50ms |
| 常见 1080p JPEG/PNG 冷缓存打开 | < 150ms |
| 首屏缩略图加载 | 20 张 2 秒内完成 |
| 大文件夹滚动 | 缓存命中时接近 60fps |
| 单窗口常规浏览内存 | 目标 < 300MB |
| 单窗口硬上限 | < 512MB |
| 空闲 CPU | < 1% |

以上指标限定在本地 SSD、非 iCloud 占位文件、非网络盘场景。外接 HDD、网络盘、iCloud 未下载文件需要显示加载状态，不承诺同等指标。

### 8.2 兼容性

- 最低系统：**macOS 15.6**
- 开发环境：**Xcode 26.5**
- 架构：Apple Silicon 优先
- 显示：Retina、多显示器、不同 DPI

### 8.3 隐私

- 默认离线运行。
- 不收集用户数据。
- 不上传图片。
- 网络访问仅限用户主动粘贴 URL 或后续版本主动触发的功能。

## 9. 分发策略

Readpic v1 采用 GitHub 开源分发，不计划申请 Apple Developer Program。

### 9.1 v1 分发方式

- GitHub Release 发布 DMG。
- 用户下载后拖入 Applications。
- 首次打开可能需要右键点击 App，选择「打开」。
- README 中明确说明未经过 Apple notarization。

### 9.2 用户预期

Readpic 面向能接受开源 macOS 软件安装方式的用户。v1 不承诺类似 App Store 或 Developer ID notarized 应用的一键安装体验。

### 9.3 后续可选

如果未来用户规模扩大，可再考虑：

- Apple Developer ID 签名
- Notarization
- Sparkle 自动更新
- Homebrew Cask

## 10. 成功标准

v1 成功标准：

- 新用户 30 秒内能完成打开图片、翻页、缩放。
- 打开 1000 张图片文件夹时 UI 不假死。
- 常见图片格式无需配置即可打开。
- 用户无需阅读文档即可找到删除、旋转、全屏、信息功能。
- 常见浏览路径比系统「预览」更顺滑。
- GitHub Release 安装说明清晰，用户能按说明完成首次打开。
